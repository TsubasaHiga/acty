# Codex コードレビュー

featureブランチの変更を Codex プラグイン（`/codex:review`）にレビュー依頼し、指摘がなくなるまで修正を繰り返します。

## 手順

1. **ブランチ状態を確認**
   - 現在のブランチが `feature/#xxx` 形式であることを確認
   - `feature/` で始まらない場合はエラーメッセージを表示して終了
   - 未コミットの変更がないことを確認（あれば警告）

2. **Codex プラグインでレビューを実行**
   - 重点ポイント（$ARGUMENTS）が指定されている場合:
     - `/codex:adversarial-review` を使用し、重点ポイントをフォーカステキストとして渡す
   - 重点ポイントが指定されていない場合:
     - `/codex:review --base main` を使用
   - Skill ツールで実行する

3. **レビュー結果をユーザーに提示**
   - Codex からの返答をそのまま表示
   - サマリー（Critical/Warning/Info/Good の件数）を強調表示
   - Critical または Warning が 0 件の場合は「レビュー通過」を通知

4. **レビュー結果の精査（重要）**
   - Codex の指摘を**鵜呑みにしない**。各指摘を深く理解し、妥当性を判断する
   - 各指摘について判断: 妥当→修正 / 妥当だが別解あり→相談 / 的外れ→スキップ
   - ユーザーに精査結果と対応方針を提示し、承認を得てから修正を実施

5. **修正・再レビューループ**
   - 承認された修正を実施
   - 修正完了後、再度 `/codex:review --base main` でレビュー
   - Critical/Warning が 0 件になるまでループ（最大3回）

6. **完了**
   - 最終レビュー結果のサマリーを表示

## 注意事項

- レビュー結果の修正はユーザー承認後に行う
- 修正時は CLAUDE.md のコーディング規約に従う
- Codex プラグインが利用不可の場合はエラーメッセージを表示して終了
